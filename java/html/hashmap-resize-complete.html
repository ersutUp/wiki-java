过程说明

HashMap 初始状态：容量为 8，阈值为 6 (8 × 0.75)。当元素数量超过阈值时，将触发扩容。

扩容核心原理

  • 触发条件：当元素数量超过阈值（容量 × 负载因子）时触发扩容
  • 容量变化：新容量 = 旧容量 × 2，始终保持为 2 的幂
  • 元素重分配：利用哈希值高位信息，无需重新计算哈希码
  • 新索引计算：要么与旧索引相同，要么 = 旧索引 + 旧容量

索引计算优化 

// 旧容量为8 (2³)，旧掩码 = 8-1 = 7 (00000111)
旧索引 = hash & 7

// 新容量为16 (2⁴)，新掩码 = 16-1 = 15 (00001111)
新索引 = hash & 15

// 新索引只有两种可能：
if ((hash & 8) == 0) {
    新索引 = 旧索引
} else {
    新索引 = 旧索引 + 8  // 旧容量
}
这种优化避免了重新计算哈希码，只需检查哈希值的第4位（8的二进制是1000）即可确定新索引
